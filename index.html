    Pocket M8

    End-to-End UX Case Study · Sozial Digital

    Role: UX/UI Designer
    Project Type: Live Product · Client Redesign
    Platform: Web Application
    Timeline: Multi-phase (Discovery → Iteration → Validation)

    Project Overview

    Pocket M8 is a digital social-support platform developed under Sozial Digital, designed to help users manage tasks, events, and communication in a sensitive, trust-driven context.

    Although the product was already live, usability breakdowns, unclear flows, and missing feedback states were impacting user confidence. My role was to audit, redesign, and structurally improve the experience while respecting real-world constraints.

    Problem Statement

    • Unclear onboarding and entry points
    • Confusing task vs event creation
    • Poor feedback states in messaging
    • No clarity when a contact became unavailable
    • Overloaded dashboards
    • Lack of emotional cues in sensitive interactions

    These issues created hesitation, repeated failed actions, and reduced trust — a critical risk for a social-support product.

    Goals & Success Criteria

    Design Goals

    • Reduce cognitive load across key flows
    • Clarify similar actions (tasks vs events)
    • Improve system feedback and error states
    • Design with emotional clarity
    • Maintain simplicity while scaling features

    Success Indicators

    • Fewer user doubts during walkthroughs
    • Clear decision-making moments
    • Reduced need for explanation
    • Stakeholder approval

    Design Process

    1. Audit & Discovery – Heuristic evaluation of the live product
    2. Problem Framing – Mapping friction points to user intent
    3. Information Architecture – Reducing clutter and grouping actions
    4. User Flow Redesign – Eliminating ambiguity and dead ends
    5. UX Writing & Microcopy – Human, calm, supportive language
    6. Validation & Iteration – Scenario testing and client walkthroughs

    The process was iterative and constraint-aware, focusing on realistic improvements rather than idealized redesigns.

    Research & Discovery

    Existing Product Audit

    • Navigation clarity
    • Content hierarchy
    • Feedback visibility
    • Error handling
    • Accessibility basics

    Stakeholder Inputs

    • Real user complaints
    • Technical constraints
    • Non-removable features
    • Long-term vision

    User Behavior Signals

    • Live product testing
    • Edge-case exploration
    • Scenario-based walkthroughs

    Key UX Challenges

    • Ambiguous Creation Flow — Tasks and events lacked distinction
    • Messaging Failures — No feedback when contact unavailable
    • Dashboard Overload — Competing elements with no hierarchy
    • Emotional Disconnect — Cold system responses

    Key Design Decisions

    Task vs Event Separation

    Tasks and events were explicitly separated with contextual descriptions to help users understand consequences before proceeding.

    Messaging Unavailable State

    • Disabled input field
    • Clear explanatory message
    • Visual icon reinforcement

    Magic Link Login

    Introduced phone-number-based magic links with clear explanations to reduce anxiety for non-technical users.

    Visual Design

    • Clean layouts
    • Soft contrast
    • Clear typography
    • Familiar patterns

    The interface prioritizes readability, accessibility, and calmness over visual noise.

    Outcomes & Impact

    • Improved clarity across workflows
    • Reduced user confusion
    • Stronger emotional alignment
    • High client satisfaction

    Learnings

    • UX in sensitive domains requires emotional intelligence
    • Silent failures damage trust
    • Microcopy prevents support tickets
    • Constraints improve realism

    Future Improvements

    • Formal usability testing
    • Accessibility audits
    • UX metrics
    • Design system documentation

    Final Note

    This project strengthened my ability to design responsibly for real users, collaborate with stakeholders, and work effectively on live products under constraints.
